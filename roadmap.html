V1.8 — Trace GPS, AIS & édition ⏳ En cours

L'objectif : avoir une trace propre en mer sans avoir à laisser le téléphone allumé pendant des jours. Plusieurs sources se complètent et s'auto-nettoient.

Principe directeur — Pour une longue traversée, garder un iPhone écran-allumé est irréaliste. La V1.8 introduit donc plusieurs sources GPS qui se relaient :
→ AIS (transpondeur du bateau, capté par satellite) pour le suivi continu en mer
→ Import GPX du calculateur de bord (B&G / Raymarine / Garmin) pour la haute fidélité
→ iPhone GPS avec verrouillage écran pour les sorties courtes (annexe, marina, à terre)
→ Marqueurs manuels 1 tap pour les évènements ("on empanne", "dauphins")

Vie privée & sécurité — la sync AIS peut être mise en pause à volonté (zones à risque, transits sensibles, contraintes d'assurance), et la diffusion sur la page publique est différée par défaut de 48 h, ou contrôlée manuellement par le skipper. La famille connectée voit toujours en temps réel.

  • Sauter à la carte : clic sur un chapitre, une étape ou un post → bouton "Voir sur la carte" qui zoome sur la zone concernée (ou pose une épingle sur le post si géolocalisé).

V3.4 — Import historique unifié (GPX + AIS Datalastic) ⭐ 💤 Si besoin (ex-V2.11)

Sprint regroupant deux specs précédemment séparées (v1.8.7 import/export GPX et v1.8.2b récupération AIS historique Datalastic). Les deux partagent la même tuyauterie — déduplication, auto-allocation à l'étape, consolidation post-import — donc on les construit en une seule fois plutôt que deux fois. Quand l'activer ? Post-départ, à la demande : la couverture AIS pendant le shake-down méditerranéen s'est révélée suffisante, ces imports deviennent un confort plutôt qu'un blocant.

Partie 1 — Import / Export GPX (ex-v1.8.7)

  • Import du calculateur de bord : zone de dépôt + sélecteur de fichier dans Réglages → Suivi GPS → "Importer / Exporter un GPX". Sélecteur d'étape (par défaut : étape couvrant la première date du fichier). Source par défaut 'gpx' ; aperçu avant import ("412 pts · 287 nouveaux · 125 doublons").
  • Déduplication intelligente : ±2 min ET haversine ≤ 50 m sur les points existants → flaggés en doublon. L'utilisateur confirme avant insert.
  • Export : choix d'une étape (ou tout), sortie .gpx. Filtre par défaut "points consolidés uniquement" (couche d'affichage), bascule "Inclure les supersédés".
  • Time zones : <time> est UTC par spec mais certains calculateurs écrivent l'heure locale. Détection + warning si les dates semblent décalées de plusieurs heures.

Partie 2 — Récupération AIS historique Datalastic (ex-v1.8.2b)

  • API Datalastic (payant, à la demande) — endpoint historique par MMSI + plage de dates. Estimé ~5 €/requête pour quelques jours selon les tarifs mi-2026.
  • UI : section pliable "Combler une période" dans Réglages → AIS. Plage de dates, sélecteur d'étape, bouton "Estimer le coût" (quote Datalastic) avant le bouton "Importer".
  • Provenance traçable : source='ais' + metadata.ais_provider='datalastic' en JSONB pour ne pas confondre avec les enregistrements en direct.
  • Confirmation explicite : action payante, le coût estimé est affiché à côté du bouton de confirmation. Aucun appel automatique.

Partie 3 — Bibliothèque partagée

  • Lib unifiée : src/lib/historic-import.js centralise déduplication, auto-allocation par étape (déléguée à la sortie de v1.8.9) et insert avec provenance. Réutilisée par le worker AIS en direct, l'importeur GPX et l'importeur Datalastic.
  • Hook consolidation (v1.8.8) : après chaque insert massif, déclencher consolidate_waypoints_for_etape(etape_id) pour que la source la plus fidèle prenne la priorité (GPX > AIS > iPhone).
  • Schéma : waypoints.metadata JSONB NULL (si aucun consommateur antérieur ne l'a déjà introduit).
